VCNode.js-бэкенд, который:
- принимает запросы от фронтенда;
- проксирует предсказания в Python-сервис модели;
- собирает и сохраняет разметку пользователей.

Основные эндпоинты (MVP):
- POST /api/predict — предсказание T_C по списку формул;
- POST /api/label   — сохранение пользовательской разметки (формула + T_C + признаки);
- GET  /api/labels  — выгрузка разметки **только для администратора** (JSON или `?format=csv`).

Авторизация и админка:
- POST /api/auth/register, POST /api/auth/login, POST /api/auth/logout, GET /api/auth/me;
- первый зарегистрированный пользователь получает роль admin;
- страница **/admin.html** — просмотр и выгрузка собранных данных (доступна только админу).

Подробнее: [docs/AUTH_AND_ADMIN.md](docs/AUTH_AND_ADMIN.md).

**Куда попадают данные разметки**  
Запросы на POST /api/label обрабатывает `labelService` и сохраняет строки в CSV-файл:
- путь по умолчанию: `backend/data/user_labels.csv` (относительно рабочей директории процесса);
- можно задать свою папку через переменную окружения: `DATA_DIR=/path/to/dir`;
- колонки: formula, Curie_TC_K, synagonia, source, comment, created_at, client_ip.

**Как протестировать разметку**
1. Запустить бэкенд из папки `backend`: `npm run dev`.
2. Открыть в браузере главную страницу (например http://localhost:3000).
3. В блоке «Добавить разметку» заполнить формулу (обязательно) и температуру Кюри (обязательно), при желании — сингонию, источник, комментарий.
4. Нажать «Отправить разметку». Должно появиться сообщение «Разметка отправлена.».
5. Проверить файл: `backend/data/user_labels.csv` — в нём должна появиться новая строка с введёнными данными.

Запуск (пример для TypeScript + ts-node-dev):
- npm install
- npm run dev
